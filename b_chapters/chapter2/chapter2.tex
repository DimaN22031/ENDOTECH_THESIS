\chapter{System Architecture and Design}

\section{Overview of the System}

The ENDOTECH system is designed as a cross-platform solution for managing inventory and sales of medical equipment. The system supports three main platforms: Windows desktop, web interface, and Android mobile devices.

The architecture of the system ensures flexibility, scalability, and ease of use for different types of users.

\section{System Architecture}

The system follows a multi-tier architecture consisting of the following components:

\begin{itemize}
    \item User Interface (UI) – provides interaction between the user and the system;
    \item Application Layer – contains the business logic of the system;
    \item Database Layer – responsible for storing and managing data.
\end{itemize}

This architecture allows the system to be modular and easily extendable.

\section{Technologies Used}

The following technologies were used in the development of the system:

\begin{itemize}
    \item Windows application – developed using modern desktop technologies;
    \item Web application – developed using HTML, CSS, and JavaScript;
    \item Android application – developed for mobile devices;
    \item Database – used for storing product, sales, and user data.
\end{itemize}

These technologies ensure compatibility across multiple platforms.

\section{System Functionality}

The system provides the following main features:

\begin{itemize}
    \item Inventory management (adding, editing, deleting products);
    \item Sales tracking and reporting;
    \item Stock level monitoring;
    \item User-friendly interface for different devices;
    \item Data synchronization between platforms.
\end{itemize}

\section{Database Design}

The database is designed to store key information such as products, categories, sales transactions, and users.

The main entities include:
\begin{itemize}
    \item Products;
    \item Categories;
    \item Sales;
    \item Users.
\end{itemize}

The database structure ensures data consistency and efficient data retrieval.

\section{Conclusion}

The designed system architecture provides a solid foundation for the development of the ENDOTECH system. It ensures scalability, flexibility, and efficient data management across all supported platforms.
